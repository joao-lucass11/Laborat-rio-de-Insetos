Do que se trata este caderno

Projeto pessoal de Banco de Dados: um sistema de gerenciamento para um laboratório de insetos, que registra o processo entre a coleta em campo e a pesquisa científica.

O projeto envolve três áreas:

BD

Modelagem do banco de dados: coletas, insetos, taxonomia e pesquisas.

POO

Lógica do sistema, estruturada em classes e objetos.

HTML/CSS

Interface de apresentação do projeto, incluindo esta página.

Base: disciplinas de POO, Banco de Dados e Projeto Integrador. Orientação: professor Matheus Rodrigues.

Do campo até a ciência

Um coletor vai a campo e captura insetos. A coleta é registrada no sistema, associada a uma localização específica. Os insetos encontrados são cadastrados e vinculados a essa coleta, e cada um recebe sua classificação taxonômica.

Pesquisadores utilizam os insetos cadastrados para realizar pesquisas. Uma pesquisa pode estudar vários insetos, e um inseto pode participar de várias pesquisas.

Por que organizar isso importa

Sem um sistema de registro, dados de coleta, catalogação e pesquisa ficam dispersos e sujeitos a perda. Cada coleta é única: local, data e condições não se repetem.

Um banco de dados estruturado permite rastrear a origem de cada inseto, sua classificação e as pesquisas em que já foi utilizado, reduzindo retrabalho e mantendo o histórico acessível.

A aplicação de POO organiza a lógica do sistema em componentes reutilizáveis, e a interface em HTML/CSS torna essas informações acessíveis a quem não trabalha diretamente com o código.
